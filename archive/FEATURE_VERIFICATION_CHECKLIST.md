## 🔍 详细测试场景

### 场景 1：完整的 QSL 卡片生命周期
1. 创建新日志
2. 为日志分配 QSL 卡号（状态：未分配 → pending）
3. 扫码出库（状态：pending → out_stock）
4. 验证状态和时间记录
5. 回收卡片（状态：out_stock → pending）

### 场景 2：连续日志记录
1. 开启连续日志模式
2. 填写第一条日志并保存
3. 验证除呼号外的字段是否保留
4. 填写第二条日志并保存
5. 验证时间是否自动更新

### 场景 3：ADIF 导入和序号重排
1. 导入 ADIF 文件
2. 验证日志是否按时间排序
3. 验证序号是否重新分配
4. 如果有 QSL 卡片，验证绑定是否正确

### 场景 4：打印流程
1. 添加 QSL 标签到打印队列
2. 添加地址标签到打印队列
3. 批量选择打印
4. 验证是否直接调用浏览器打印对话框
5. 打印完成后清空队列
6. 验证 temp 文件夹是否清空

### 场景 5：Python 数据库导入
1. 准备 Python 版本的数据库文件
2. 通过设置页面上传
3. 验证导入进度提示
4. 验证数据是否正确导入
5. 验证 QSL 状态转换是否正确
6. 验证出入库时间是否保留

### 场景 6：地址自动填充
1. 在地址库中添加地址
2. 设置默认发卡地址
3. 打印地址标签时输入呼号
4. 验证地址是否自动填充
5. 验证发卡地址是否自动填充

## 🐛 已知问题

### 1. SQL.js 参数类型问题 ✅ 已修复
- 问题：datatype mismatch 错误
- 解决：修改 sqljs-adapter.js，所有参数使用 parseInt()

### 2. 数据库 null 记录 ✅ 已修复
- 问题：callsigns 表中有 null 记录
- 解决：添加 WHERE callsign IS NOT NULL 过滤

### 3. 设置页面白屏 ✅ 已修复
- 问题：设置主呼号后页面白屏
- 解决：改用本地状态更新，避免不必要的数据重新获取

### 4. 重置后数据未清除 ✅ 已修复
- 问题：重置 QSL 数据后 Dashboard 仍显示旧数据
- 解决：重置成功后自动刷新页面
